# 第一部分：审稿意见（以 AAAI 资深审稿人视角）

下面先以一个挑剔的 AAAI 审稿人视角，对当前 M2NA 方案做一次严格 review。然后第二部分给出修订、扩充后的完整思路。

## A. 总体评价

**Strengths（认可的部分）**
- 任务收窄合理：从"寓言生成"收窄到"机制图→隐性叙事类比"是正确的方向，避免了 creative writing 的模糊评估。
- 形式化清晰：`(c, G_c, T_c) → (N, A)` 的输入输出定义是可操作的。
- 评估有层次：自动+专家+人类理解三层有说服力。
- 相关工作扎实：2024-2026 文献覆盖到位，gap 陈述合理。

**Weaknesses（如果我是审稿人，会打 reject/borderline 的理由）**

下面是七个最致命的问题，必须在投稿前解决。

---

## B. 七个致命问题（Major Concerns）

### B1. 「机制图 G_c 是输入」会被质疑：任务被简化成了 graph-to-text

**问题**：如果 G_c 作为输入直接给模型，那这篇论文很大程度上退化成一个"带约束的 graph-to-text"任务。审稿人会问：
- 机制图谁构造的？如果是人工/LLM 预先构造好，那真正难的部分（概念→结构）被你藏在数据集构建里了，模型只是在做填空。
- 这跟 PlanGTG (NAACL 2025) 的区别只剩"换了个域 + 禁用词"，创新不足。

**后果**：贡献被压缩成"graph-to-text + 词表过滤 + 换皮"。

**必须的修复**：见第二部分 C1——引入**两种任务设定**（G_c given vs. G_c not given），把"概念结构抽取"也纳入评测，让任务真正端到端。

---

### B2. 评估的「循环论证」风险：用 LLM 生成数据，又用 LLM judge 评估

**问题**：
- 数据集 gold narrative 用 LLM (StructNarr) 生成，主实验里 StructNarr 又是 ours，自动评估又用 LLM judge。
- 审稿人会直接质疑：你的方法赢，是因为它和数据生成/评估用了同源偏好。这是 benchmark 论文的头号杀手。

**后果**：所有"ours 最好"的结论都不可信。

**必须的修复**：见 C4——
1. Gold set 必须有**人类撰写或人类大改写**的部分，不能全是 LLM 自产自评。
2. LLM judge 必须做**与人类评分的相关性验证**，且 judge 模型与被评模型解耦（不要用 GPT-4 judge GPT-4 的输出）。
3. 引入**不依赖 LLM 的硬指标**（leakage 用词表+NLI，coverage 用 entailment 模型）。

---

### B3. 「Coverage / Mapping Consistency」的度量太软，缺少可复现的算子

**问题**：
- `Concept Coverage = Σ element_score / 2m`，但 element_score (0/1/2) 由 LLM judge 主观给出，无法复现，也无法被其他研究者复用。
- Mapping Consistency 的四个子维度（Existence/Correctness/Specificity/Non-spuriousness）没有可操作定义。

**后果**：benchmark 不可复现 = 不是合格 benchmark。

**必须的修复**：见 C3——给每个指标一个**带 anchor 的 rubric + 参考实现**，并提供 NLI-based 的自动近似，报告它与人评的相关性。

---

### B4. 缺少「人类基线 / 上界」——不知道任务到底有多难

**问题**：现在所有 baseline 都是 LLM。没有人类写的隐性叙事类比作为 upper bound，审稿人无法判断：
- 当前 SOTA 模型离人类有多远？
- 这个任务对人类是否也难（说明是真问题）还是 trivial（说明不值得研究）？

**必须的修复**：见 C2——加入 **Human upper bound**（专家写的 N+A）和 **Human lower bound**（普通人不给指导写的），把模型放在人类区间里看。

---

### B5. 「为什么必须是隐性叙事？」的动机论证不够硬

**问题**：审稿人最爱问的反问——
- 隐藏术语到底有什么用？直接给定义+类比不就行了？
- "implicitness" 是不是为了难而难（artificial difficulty）？

当前方案靠"人类靠故事学习"这种 hand-wavy 的动机，不足以撑住一篇 AAAI。

**必须的修复**：见 C5——把动机从"故事更好记"升级为一个**可检验的科学假设**：
> 词汇隐藏迫使模型进行真正的结构迁移（structural alignment），而非术语复述（lexical shortcut）。因此 M2NA 可以作为探测 LLM 是否具备 relational/analogical reasoning 的诊断任务。

这样论文从"教育应用"升级为"LLM 类比推理能力的诊断 benchmark"，AAAI 接受度更高。

---

### B6. 任务可能存在「评估冲突」：高覆盖 vs 强隐藏是矛盾的

**问题**：覆盖机制要表达清楚，但又要隐藏术语、保持最小性，这几个目标天然冲突。当前方案没有：
- 定义如何聚合多个冲突指标（是不是 Pareto？还是加权？）
- 一个统一的主指标让方法可比较、可排序。

**后果**：表格里每个方法各有高低，得不出"谁更好"的结论，论文没有 punch line。

**必须的修复**：见 C3.4——定义一个**复合主指标 M2NA-Score**（覆盖 × 隐藏 × 映射的调和/几何平均，配合 gate），让结果可排序，同时仍报告分项。

---

### B7. 人类理解验证（Layer 3）设计有内生缺陷

**问题**：
- "读隐性叙事后回答理解题"——但隐性叙事故意隐藏概念名，被试可能根本不知道这讲的是"路径依赖"，怎么答"请解释路径依赖"？
- between-subject per concept + 30-50 人，每组 n 太小，统计功效不足。

**必须的修复**：见 C6——
1. 理解测试要分两阶段：先读叙事→做迁移/识别题（不需要知道术语），再揭示术语→做定义题。
2. 给出 power analysis 估算最少样本量，或老实降级为"preference + comprehension 的 pilot study"，并明确标注非主结论。

---

## C. 其余中等问题（Minor / 需要补强）

- **C-a 命名**：StructNarr 平淡。建议方法名突出"对齐先行"，如 **AlignFirst** 或 **PALN (Plan-Align-then-Narrate)**。
- **C-b 难度标签**：difficulty 的 easy/intermediate/hard 由谁定？需要一个可操作的难度定义（如机制图节点数 + 跨域距离）。
- **C-c 语言**：中英双语会稀释精力。建议**英文为主**，中文做小规模 case study 验证跨语言泛化即可。
- **C-d 去重/污染**：概念取自 Wikipedia，模型预训练见过——需要讨论 data contamination，并强调任务难点在"隐藏术语下重构结构"而非"记忆定义"。
- **C-e 关系类型**：E_c 的关系标签（enables/increases/causes...）需要一个封闭集合（schema），否则标注不一致。
- **C-f 错误分析**：缺少 failure taxonomy（泄露型、机制缺失型、虚假映射型、模板型），这是 benchmark 论文的加分项。
