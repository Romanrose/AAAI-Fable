## 一、Teaching Gradient Descent Through Analogies, Step by Step（TU Delft, 2025）

**性质**：TU Delft EEMCS **本科毕业论文**（CSE3000 Research Project，作者 Thomas Koppelaar）。**不是 peer-reviewed 会议论文，规模很小、结论为 null result**——威胁等级低，但任务方向高度相似，需引用区分。

### 任务与研究问题
- RQ：用类比解释 Gradient Descent，对 CS 学生的学习成效有何影响？
- 子问题：SQ1 专家如何评类比；SQ2 学生用类比学到什么；SQ3 学生参与感如何。
- 理论框架：Notional Machines（计算教育里的"概念机器"），把 analogy 当作其中一类。

### 概念与类比怎么来
- 概念来源：Goodfellow《Deep Learning》+ Bishop《PRML》两本教科书，抽 6 个 GD 相关概念：A1 优化与损失函数 / A2 梯度 / A3 梯度下降 / A4 临界点 / A5 批量GD / A6 随机GD。
- 类比：**全部手工编写**（理由：历史上 Notional Machines 多为手工）。统一用一个"盖革计数器在辐射区找安全地带"的源域贯穿所有概念。
- 每个类比配一张 **concept map 表**（概念属性 ↔ 类比对应物），辅助专家评审——这其实就是一种弱版 alignment。

### 专家评估指标（直接和你的相关）
三个 1–3 李克特维度（1低/2中/3高），定义来自 Bhavya 等的 analogy evaluation criteria：
- **TCC (Target Concept Coverage)**：类比覆盖概念描述里多少要点。≈ 你的 Concept Coverage。
- **MS (Mapping Strength)**：源↔目标对应的逻辑严谨与一致性。≈ 你的 Mapping Consistency。
- **M (Metaphoricity)**：源域和目标的概念距离（越远越"隐喻"）。
- 样本：15 名合格专家（1 教授 + 1 硕士 + 3 TA + 10 本科已修ML），每个类比平均仅被评 4.67 次。
- **一致性差**：Krippendorff α 多为负值（系统性分歧），只有 GD 那条 α>0。作者归因于样本小 + 任务本身主观。

### A/B 真人学习实验（值得借鉴的设计）
- 被试：**15 名没修过 ML 的 CS 新生**（特意选 novice，引 Treagust：已达 formal operational 的人用类比反而冗余）。
- 设计：随机分 A(对照)/B(实验)。每个概念：先做选择题(含"我不知道"选项) → 给定义(B组再加类比) → 再做选择题。**knowledge gain = 后测分 − 前测分**。
- 学习目标锚定 Bloom 修订版的 Understand 层（不测 Apply）。
- 末尾用精简版 RIMMS（5 题）测 Attention/Relevance/Confidence/Satisfaction。

### 结果（关键）
- **knowledge gain 无统计显著差异**（独立样本 t 检验：优化 p=0.778；GD p=0.497）。
- **engagement 也无显著差异**。
- 作者解释：样本太小 + 任务主观 + 调查问卷是单向信息流、缺乏互动（引文献说"主动参与的类比教学才有效"）。

### 对本文的启示 / 区分点
- **区分点（写进 Related Work）**：它单一源域手工造类比、6 个概念、无 benchmark、**无词汇隐藏、无机制图作为输入、无显式 alignment 作为输出物、无大模型生成**。
- **它替你踩的坑**：(1) 专家评 analogy 主观性极高、一致性会很差 → 你必须设计更客观的 rubric + 锚点示例 + 报告 IAA；(2) 单向问卷 + 小样本测不出显著 → 你的最小真人实验要么加大样本、要么改成更敏感的设计（如交互/迁移题）；(3) 选 novice 被试这一点做得对，可直接沿用。
- **它给你的口子**：作者把 future work 明确留给"AI 能否生成比人工更高质量的类比"——正是你的切入点。
- 链接：https://repository.tudelft.nl/file/File_41a64931-d59d-4574-a3a4-05cb41af9ca1

## 二、Teaching Through Analogies: A Modular Pipeline for Educational Analogy Generation（arXiv 2605.24211, 2026）

**性质**：正式 arXiv 论文，跨 12 个 LLM（6 家族）+ 7 个 embedding 模型的系统评估。**方法 pipeline 的真正先到者**。

### 任务与四阶段 pipeline（≈ 你们的 KG-Fable）
基于 Structure Mapping Theory，拆成：
1. **Source Finding (RQ1)**：给目标概念找源系统，分 closed（从池子检索）/ open（自由生成）。
2. **Sub-concept Generation (RQ2)**：生成/匹配源↔目标的组件映射。
3. **Explanation Generation (RQ3)**：生成连接类比的自然语言解释。
4. **Evaluation (RQ4)**：LLM judge + 人工标注评质量。

### 数据集（你可能能直接复用）
- **SCAR**：400 个科学类比实例，13 领域，**1,614 条标注的 sub-concept 映射** + 背景描述 + 解释。
- **ParallelPARC**：310 个 gold 类比，来自科学过程描述，标注为 **(entity, verb, entity) 的关系对齐**。
- → 这两个数据集的"sub-concept 映射 / 关系对齐"结构，和你的 G_c + alignment A 高度同构，可作为概念来源或 baseline 数据。

### 评估指标
- **Hit@K**：top-K 检索里是否≥1 个 gold 源（二值）。
- **System Accuracy**：所有 sub-concept 对全配对正确才得 1。
- **Semantic Match Accuracy (SMA)**：count(相似度≥0.7)/总数，用 all-MiniLM-L6-v2，cos 阈值 0.7。
- **解释质量**：对 gold 参考算 cosine（all-mpnet-base-v2）。
- **Semantic Hit@K**：生成源与任一 gold 源 embedding 相似度 >0.374（上三分位阈值）即算对。

### LLM-as-judge 设置（直接可借）
- 评委：**Claude Sonnet 4.6**，独立打 1–3 分三维度：Coherence（配对是否直觉合理）/ Mapping Soundness（结构性对应）/ Explanatory Power（是否帮学习者理解）。
- 用 **10 个跨分数段的校准 few-shot 示例**稳定打分。

### 主要发现（关键，写动机/对照都用得上）
- **Sub-concept grounding 大幅提升解释质量**：平均 cosine 0.724（只给系统）→ 0.854（unpaired sub-concept）→ 0.830（paired）。背景描述几乎没用（≤0.006）。→ 印证"结构化中间表示有用"，是你 G_c 的间接证据。
- 开放式找源域，加 sub-concept 仅提升 0.4–1.6 个点；顶级模型挤成一团（Gemini-2.5 75.4% / GPT-OSS-120B 75.3% Hit@20）。
- Sub-concept 匹配：Grok-4 达 0.78 system accuracy（超过此前 SOTA 0.62）；**加背景描述反而降分**（0.603 vs 0.650）。
- **LLM-judge 更适合做比较排序而非绝对打分**：Claude 与最强标注者的排序 Spearman ρ=0.705；标注者间一致性只到 fair（α≈0.34–0.48）。→ 你设计自动评估时，宜走 pairwise/ranking 而非绝对分。

### 作者承认的局限
- 仅英文；0.7 SMA 阈值是启发式拍的、未实证校准；**无端到端 pipeline 评估**（误差跨阶段累积未测）；单次评估无方差；ParallelPARC 只用在检索阶段；标注者来自单一机构。

### 对本文的启示 / 区分点
- **区分点（写进 Related Work）**：它做**显式**类比解释，**不藏术语、不以机制图 G_c 为输入、不要求 alignment 作为受评输出物、不做真人学习实验**。它的"sub-concept"是辅助信号，不是被禁/被对齐的对象。
- **可复用**：SCAR / ParallelPARC 数据；LLM-judge 的"校准 few-shot + 排序优于绝对分"经验；"结构化中间表示提升质量"作为你 G_c 必要性的旁证。
- 链接：https://arxiv.org/html/2605.24211v1

## 三、两篇合起来对本文定位的结论

1. **任务侧**：TU Delft 证明"ML 概念类比 + 真人 A/B"这条路有人走过，但做得浅（本科、手工、null result）。你的增量必须是 **benchmark 化 + concealment + 机制图输入 + 显式对齐 + 更可信的真人实验**。
2. **方法侧**：Teaching Through Analogies 占了"拆结构再生成"的 pipeline。你的护城河只能落在它没碰的三件事：**词汇隐藏、机制图作为输入、alignment 作为受评输出**。
3. **评估侧**：两篇都暴露了"analogy 质量主观、标注者一致性差"的通病 → 你要么把指标做得更客观（覆盖/泄露/对齐可程序化判定），要么走 pairwise ranking + 报告 IAA + 自动指标与人工相关性。
4. **共同空白 = 你的全部价值**：没有人在生成时**刻意隐藏目标概念术语**。这一条务必在 Intro 第一段就立起来。
