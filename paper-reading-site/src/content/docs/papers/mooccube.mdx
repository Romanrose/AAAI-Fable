# MOOCCube: A Large-scale Data Repository for NLP Applications in MOOCs

**年份/来源**：2020 · ACL  
**所属支柱**：[知识图谱](/pillars/knowledge-graph/)  
**PDF**：[打开本地 PDF](/papers/KG-01-MOOCCube-ACL-2020.pdf)  
**原文链接**：[https://aclanthology.org/2020.acl-main.285/](https://aclanthology.org/2020.acl-main.285/)

## 这篇论文在讲什么

构建面向 MOOC 场景的大规模教育数据仓库。

## 与 Concept-to-Fable 的关系

它展示了教育资源、概念和学习行为如何被组织成可计算资源。

## 推荐阅读部分

重点看数据 schema、概念资源和教育 NLP 应用方式。

## 可借鉴点

可借鉴数据组织方式，为小学知识点、教材段落和生成寓言建立统一索引。

## 局限与注意事项

MOOC 场景与小学全学科教材不同，且不面向寓言生成。

## 研究导向 Q&A

### 1. 这篇论文解决的问题和 Concept-to-Fable 有什么关系？

它展示了教育资源、概念和学习行为如何被组织成可计算资源。

### 2. 它支撑四大支柱中的哪一部分？

它主要支撑 **知识图谱**。在 Concept-to-Fable 中，这一部分的任务是让模型先理解课程知识结构，再决定寓言应该解释什么。

### 3. 我们可以借鉴它的什么方法、数据或评测方式？

可借鉴数据组织方式，为小学知识点、教材段落和生成寓言建立统一索引。

### 4. 它没有解决什么，因此我们的任务还有什么研究空间？

MOOC 场景与小学全学科教材不同，且不面向寓言生成。 这也正是 Concept-to-Fable 可以继续推进的空间：把论文中的单点能力放进“课程 KG → 结构映射 → 寓言生成 → 教育评价”的完整链路里。

### 5. 如果把它用于“知识点转寓言”，需要怎样改造？

需要转换到 K-12 课程标准，并补充寓言生成所需的映射与故事字段。
