# A Survey on Automatic Generation of Figurative Language

**年份/来源**：2024 · ACM Computing Surveys  
**所属支柱**：[结构映射](/pillars/structure-mapping/)  
**PDF**：[打开本地 PDF](/papers/Figurative-01-FigurativeLanguageGenerationSurvey-ACMCSUR-2024.pdf)

## 这篇论文在讲什么

综述比喻、隐喻等修辞语言自动生成。

## 与 Concept-to-Fable 的关系

寓言生成依赖修辞和跨域表达，这篇能提供语言层面的背景。

## 推荐阅读部分

重点看 figurative language 的类型、生成方法和评价。

## 可借鉴点

可用于说明 Concept-to-Fable 与 metaphor/analogy generation 的区别。

## 局限与注意事项

修辞生成通常不要求完整教学闭环。

## 研究导向 Q&A

### 1. 这篇论文解决的问题和 Concept-to-Fable 有什么关系？

寓言生成依赖修辞和跨域表达，这篇能提供语言层面的背景。

### 2. 它支撑四大支柱中的哪一部分？

它主要支撑 **结构映射**。在 Concept-to-Fable 中，这一部分的任务是把抽象知识结构稳定映射成寓言中的角色、冲突和情节。

### 3. 我们可以借鉴它的什么方法、数据或评测方式？

可用于说明 Concept-to-Fable 与 metaphor/analogy generation 的区别。

### 4. 它没有解决什么，因此我们的任务还有什么研究空间？

修辞生成通常不要求完整教学闭环。 这也正是 Concept-to-Fable 可以继续推进的空间：把论文中的单点能力放进“课程 KG → 结构映射 → 寓言生成 → 教育评价”的完整链路里。

### 5. 如果把它用于“知识点转寓言”，需要怎样改造？

需要把修辞表达与课程知识、故事结构和学习目标绑定。
