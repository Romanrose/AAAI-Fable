# A Survey of Controllable Text Generation using Transformer-based Pre-trained Language Models

**年份/来源**：2022 · arXiv  
**所属支柱**：[层次化生成](/pillars/hierarchical-generation/)  
**PDF**：[打开本地 PDF](/papers/StoryGen-05-ControllableTextGenerationSurvey-arXiv-2022.pdf)  
**原文链接**：[https://arxiv.org/abs/2201.05337](https://arxiv.org/abs/2201.05337)

## 这篇论文在讲什么

综述 Transformer 时代的可控文本生成方法。

## 与 Concept-to-Fable 的关系

它支撑生成端控制策略选择，例如属性控制、规划控制和解码控制。

## 推荐阅读部分

重点看不同控制方法的输入形式、控制粒度和代价。

## 可借鉴点

可用于选择控制教学目标、年级难度、寓言风格的技术路线。

## 局限与注意事项

综述不专门讨论故事结构和教育准确性。

## 研究导向 Q&A

### 1. 这篇论文解决的问题和 Concept-to-Fable 有什么关系？

它支撑生成端控制策略选择，例如属性控制、规划控制和解码控制。

### 2. 它支撑四大支柱中的哪一部分？

它主要支撑 **层次化生成**。在 Concept-to-Fable 中，这一部分的任务是把生成过程拆成可控步骤，避免 LLM 直接写作时跑偏。

### 3. 我们可以借鉴它的什么方法、数据或评测方式？

可用于选择控制教学目标、年级难度、寓言风格的技术路线。

### 4. 它没有解决什么，因此我们的任务还有什么研究空间？

综述不专门讨论故事结构和教育准确性。 这也正是 Concept-to-Fable 可以继续推进的空间：把论文中的单点能力放进“课程 KG → 结构映射 → 寓言生成 → 教育评价”的完整链路里。

### 5. 如果把它用于“知识点转寓言”，需要怎样改造？

需要从通用可控生成中筛选适合长文本寓言和知识约束的方法。
